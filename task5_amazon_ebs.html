Terraform Configuration

The main.tf file defines the AWS infrastructure, including a VPC, subnet, security group, EC2 instance, and an EBS volume to be attached.

Terraform Initialization

The `terraform init` command initializes the project, downloading the necessary AWS provider plugin.

Terraform Validation

Running `terraform validate` checks the configuration files for syntax errors, confirming the code is valid.

Terraform Plan

The `terraform plan` command shows the execution plan, detailing the resources to be created, such as the EBS volume and EC2 instance.

Terraform Apply

The `terraform apply` command executes the plan, provisioning the defined resources in the AWS cloud.

EC2 Instance Created

The AWS Management Console shows the newly created EC2 instance, 'EBSDemoInstance', is running successfully.

EBS Volumes Created

The console displays two volumes: the instance's root volume and the additional 'MyEBSVolume' created via Terraform.

EBS Volume Details

A detailed view of the root EBS volume, showing it is 'in-use' and attached to our EC2 instance.

Terraform Destroy Plan

Running `terraform destroy` generates a plan to tear down all the resources managed by this configuration.

Infrastructure Destroyed

After confirmation, Terraform destroys all created resources, cleaning up the environment. The output confirms 6 resources were destroyed.
