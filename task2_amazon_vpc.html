1. Terraform Configuration

The main.tf file defines the desired infrastructure, including the AWS provider, data sources for an Ubuntu AMI and the default VPC, and resources like a security group.

2. Initialize Terraform

The terraform init command prepares the working directory by downloading necessary provider plugins (e.g., hashicorp/aws) and creating a lock file to ensure consistent provider versions.

3. Validate Configuration

Running terraform validate checks the syntax of the configuration files. A "Success!" message confirms that the code is syntactically correct and ready for planning.

4. Create an Execution Plan

The terraform plan command shows what actions Terraform will take to achieve the desired state. Here, it indicates that it will create one EC2 instance and other associated resources.

5. Apply the Changes

terraform apply executes the plan. After user confirmation, Terraform provisions the resources in AWS, creating the EC2 instance and security group as defined in the code.

6. Verify in AWS Console

After a successful apply, the newly created EC2 instance is visible in the AWS Management Console, showing its status as "Running" and "Initializing."

7. Instance Summary

The instance summary provides detailed information, confirming its public IP address, instance type (t2.micro), and that it has been launched in the correct VPC.

8. Plan the Destruction

The terraform destroy command creates a plan to remove all managed infrastructure. The output lists every resource that will be deleted upon confirmation.

9. Destroy the Infrastructure

After confirming the destruction plan, Terraform proceeds to delete all the created resources. The output confirms that 7 resources were successfully destroyed, cleaning up the environment.
