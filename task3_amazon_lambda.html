⚙️ Task 3: Amazon Lambda

1. Lambda Terraform Configuration

The main.tf file defines the IAM role, policy attachment, archive file for the code, and the AWS Lambda function resource itself.

2. Initialize Terraform

The terraform init command prepares the working directory by downloading the necessary AWS and archive provider plugins.

3. Validate Configuration

Running terraform validate checks the configuration for syntax errors, confirming that the code is valid and ready to be planned.

4. Plan the Deployment

terraform plan shows the execution plan, detailing the resources that will be created, such as the IAM role and the Lambda function.

5. Apply the Configuration

terraform apply executes the plan, provisioning the Lambda function and its associated IAM role in the AWS cloud after confirmation.

6. Verify Function in AWS Console

After deployment, the hello_lambda function appears in the AWS Lambda console, confirming its successful creation.

7. Review Lambda Code Source

The AWS console shows the Python code that was deployed, allowing for verification of the function's logic and content.

8. Check Runtime Properties

The function's configuration details confirm the runtime is set to Python 3.11, along with other properties like architecture and handler name.

9. Plan Infrastructure Destruction

The terraform destroy command generates a plan to remove all managed resources, listing the Lambda and IAM role to be deleted.

10. Execute Destruction

After confirmation, Terraform tears down the infrastructure. The "Destroy complete!" message confirms all resources have been removed.
